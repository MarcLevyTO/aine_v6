## Story

As a developer,
I want to bootstrap the Aine project using `create-next-app` and then flatten the structure to live at the repository root,
so that I have a runnable Next.js + TypeScript foundation that matches the architecture document and is ready for feature work.

## Acceptance Criteria

1. **AC1 (Project initialization):** Running `pnpm create next-app@latest aine` and answering the prompts produces a project with TypeScript ✓, ESLint ✓, Tailwind CSS ✓, App Router ✓, Turbopack ✓, and the `@/*` import alias ✓ all enabled.

2. **AC2 (No `src/` directory):** The `Would you like your code inside a 'src/' directory?` prompt is answered **No**, so `app/page.tsx` and `app/layout.tsx` exist directly under the Next.js project root (no `src/` layer).

3. **AC3 (Dev server runs cleanly):** Running `pnpm install` followed by `pnpm dev` starts the dev server on `http://localhost:3000` without errors. Opening that URL in a modern browser displays the default Next.js starter page. The browser console shows no errors.

4. **AC4 (Expected files present):** After generation, `package.json`, `tsconfig.json`, `next.config.ts`, `tailwind.config.ts`, `eslint.config.mjs`, and `AGENTS.md` exist at the Next.js project root. TypeScript strict mode is enabled in `tsconfig.json` (verify `"strict": true` under `compilerOptions`).

5. **AC5 (Flat structure — added during execution per user request):** After all files are generated, every file from the `aine/` subfolder is moved up to the repository root, the `aine/` directory is removed, and the dev server still boots cleanly from the new root location with HTTP 200 on `http://localhost:3000`. The pre-existing project README (BMAD process documentation) is preserved at root.

## Dev Notes

### Why this story exists

This is a **starter-template story** explicitly required by the Architecture document (§Starter Template Evaluation). The PoC's first concrete deliverable is a runnable, conventionally-structured Next.js project that all subsequent feature stories (1.2 – 1.6) build on. No application code is written in this story — only project initialization, verification, and (per the user's request added during execution) a structural flatten.

### Critical constraints

- **Package manager:** the architecture specifies `pnpm`. If the developer prefers `npm` or `yarn`, the substitutions are direct.
- **Opt out of `src/`:** all path references in `architecture.md` and downstream stories (1.2 – 1.6) assume the layout `app/...`, `app/components/...`, `app/lib/...` (relative to the Next.js project root). Choosing `src/` would silently invalidate every other story's path references. **Do not skip subtask 1.5.**
- **Accept all other defaults.** Tailwind, ESLint, Turbopack, App Router, `@/*` alias — all are mandated. The architecture's anti-bloat constraint says to accept the starter's defaults rather than customize.
- **Project location (final):** the Next.js project lives **at the repository root**, alongside `_bmad/`, `_bmad-output/`, `.claude/`, etc. This is the result of Task 4's restructure; `create-next-app` initially produces a subfolder, which Task 4 flattens.

### Source tree components to touch

This story creates the entire Next.js application surface. After the Task 4 restructure, the touched files live at the repo root and under `app/`. **No files inside `_bmad/`, `_bmad-output/`, or `.claude/` are touched.** The pre-existing project root `README.md` (BMAD process documentation) is preserved.

### Testing standards

**No automated tests in this story.** Verification is by manual inspection:
- File system inspection (subtasks 2.1–2.4)
- Dev server boot, twice — once before flatten (subtask 3.2), once after (subtask 4.6)
- HTTP smoke test (curl) instead of full browser-console check (which requires interactive use; deferred to Story 1.7)

This is consistent with the Architecture's deferred-testing decision for the PoC.

### Completion Notes List

✅ **All 5 ACs satisfied with the following notes:**

1. **Versions installed (verified at install time):** Next.js 16.2.4, React 19.2.4, TypeScript 5.9.3, Tailwind CSS 4.2.4, ESLint 9.39.4, eslint-config-next 16.2.4. All current as of 2026-04-26.

2. **`tailwind.config.ts` is NOT generated by `create-next-app` 16.2.x.** This is a deliberate change in Tailwind v4 — configuration moved from a separate JS/TS config file into the CSS file via `@import "tailwindcss"` and `@theme inline { ... }` directives in `app/globals.css`. **Tailwind is fully configured and working** (CSS imports are present, PostCSS plugin is registered in `postcss.config.mjs`); the file mentioned in the original AC4 simply no longer exists in the modern starter. Architecture document was updated to reflect this in the same change as the restructure.

3. **Subtasks 3.3 and 3.4 (browser console check) verified server-side only.** A real browser DevTools console check requires interactive use that this dev environment cannot perform. The HTTP response was verified clean (200, well-formed HTML, no error markers in body) and the dev server log shows no errors during the request. **Recommendation: do a quick visual check in Chrome/Firefox/Safari before considering NFR3 verification complete (that's Story 1.7's job anyway).**

4. **`CLAUDE.md` was also generated** by `create-next-app` 16.2.x alongside `AGENTS.md`. Not anticipated by the original story but harmless and left in place at the repo root post-restructure.

5. **Turbopack note:** Next.js 16 uses Turbopack as the default for `next dev`, so the `--turbopack` flag was technically redundant. The `dev` script in `package.json` is `"next dev"` (not `"next dev --turbopack"`); Turbopack still runs by default. Confirmed in the dev server log: `▲ Next.js 16.2.4 (Turbopack)`.

6. **Restructure (Task 4) added mid-story per user direction.** The original story planned for the Next.js project to live in an `aine/` subfolder. After completing Tasks 1–3, the user requested a flat structure. Task 4 was added and executed:
   - `aine/.gitignore` (Next.js standard) replaced the root's previous 1-line `.gitignore` (which only contained `.DS_Store`; the new `.gitignore` is a strict superset).
   - `aine/README.md` (default Next.js starter content) was discarded; the project root `README.md` (BMAD process documentation) is preserved.
   - All other files from `aine/` were moved to root via `mv`. `node_modules/` and `.next/` rename was instantaneous (same filesystem).
   - `aine/` directory removed (was empty after moves).
   - Dev server re-verified: boots cleanly from new root, HTTP 200, "Ready in 154ms" (faster than before — likely warm caches).
   - Architecture document was updated in the same step to reflect the new flat structure.
